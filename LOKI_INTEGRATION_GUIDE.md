# Loki Autonomous Security Engine Integration Guide

## Overview

Loki is now fully integrated into KTOX_Pi as a complete autonomous security testing framework. Similar to how RaspyJack integrates Ragnar, Loki runs as a dedicated service with a comprehensive WebUI while being controlled through the main KTOX menu.

## Architecture

### Components

```
KTOX_Pi (Main Controller)
    ↓
Offensive Menu → Loki Engine
    ↓
loki_engine.py (Process Manager)
    ↓
Headless Loki Service (Port 8000)
    ↓
WebUI Dashboard + REST API
```

### Data Directories

```
/root/KTOx/loot/loki/
├── logs/
│   └── loki.log          # Main service log
├── output/
│   ├── crackedpwd/       # Cracked passwords and hashes
│   ├── datastolen/       # Exfiltrated data
│   ├── zombies/          # Compromised system info
│   └── vulnerabilities/  # Scan results
├── input/                # Input data for attacks
└── netkb.csv             # Network knowledge base
```

### Installation Directory

```
/root/KTOx/vendor/loki/
├── payloads/user/
│   ├── reconnaissance/
│   ├── exploitation/
│   └── exfiltration/
├── actions/              # Attack modules
├── config/               # Configuration files
├── lib/
│   ├── pagerctl.py       # KTOx LCD shim
│   └── ...vendor libs
└── ktox_headless_loki.py # Launcher script
```

## Usage

### From KTOX Menu

1. **Main Menu** → **Offensive** → **Loki Engine**
2. System checks if Loki is installed
3. If not installed:
   - Shows prompt to install
   - Downloads from GitHub
   - Displays installation progress
   - Validates installation
4. Once running:
   - Shows WebUI URL
   - Displays status (INSTALLING, RUNNING, STOPPED)
   - Provides controls (start/stop)
   - Exit to keep service running

### Installation Process

**First Time:**
```
Loki Engine
    ↓
Check installed? No
    ↓
Prompt: Install Loki?
    ↓
[If Yes]
    ↓
Update apt cache
    ↓
Clone from GitHub
    ↓
Create data directories
    ↓
Write pagerctl.py shim
    ↓
Generate headless launcher
    ↓
Verify files
    ↓
Installation complete!
```

**Subsequent Times:**
```
Loki Engine
    ↓
Check installed? Yes
    ↓
Check running?
    ├─ Yes → Show status & controls
    └─ No  → Offer to start
```

## Installation Details

### Dependencies

**System Packages:**
- nmap (network scanning)
- python3-pil (PIL/Pillow, imaging)
- git (repository cloning)

**Python Packages:**
- Flask (WebUI)
- paramiko (SSH operations)
- cryptography (encryption)
- And others (auto-installed)

### Repository

- **Source**: https://github.com/pineapple-pager-projects/pineapple_pager_loki
- **Clone Location**: `/root/KTOx/vendor/loki/`
- **Branch**: main (latest)
- **Depth**: 1 (shallow clone for speed)

### Pagerctl Shim

A custom `pagerctl.py` is generated that bridges Loki's display needs to KTOx's LCD driver:

- **Source**: Generated by `_write_shim()`
- **Location**: `/root/KTOx/vendor/loki/lib/pagerctl.py`
- **Purpose**: Provides Pager class for LCD operations
- **Compatibility**: Supports headless operation (no LCD)

### Headless Launcher

Generated launcher script customized for KTOx:

- **Source**: Generated by `_write_launcher()`
- **Location**: `/root/KTOx/vendor/loki/ktox_headless_loki.py`
- **Purpose**: Starts Loki service with proper environment
- **Environment Variables**:
  - `LOKI_DATA_DIR`: Data storage path
  - `BJORN_IP`: Device IP address
  - `LOKI_PID_FILE`: PID file location

## Configuration

### Default Settings

```python
LOKI_PORT = 8000                    # WebUI port
LOKI_REPO = "https://github.com/pineapple-pager-projects/pineapple_pager_loki"
VENDOR_DIR = /root/KTOx/vendor/loki
LOKI_DATA = /root/KTOx/loot/loki
LOKI_PID = /root/KTOx/loot/loki.pid
```

### Customization

Edit `loki_engine.py`:
```python
LOKI_PORT = 8000  # Change port if needed
```

Then reinstall Loki to apply changes:
```bash
rm -rf /root/KTOx/vendor/loki
# Re-run installation through menu
```

## Troubleshooting

### Installation Issues

**Problem**: "Install failed"
```
Solution:
1. Check internet connection
2. Verify git is installed: apt-get install git
3. Check disk space: df -h
4. Try manual clone: git clone https://github.com/pineapple-pager-projects/pineapple_pager_loki /root/KTOx/vendor/loki
```

**Problem**: "Missing pagerctl shim"
```
Solution:
1. Reinstall: rm -rf /root/KTOx/vendor/loki && [install again]
2. Manually create: /root/KTOx/vendor/loki/lib/pagerctl.py
```

### Startup Issues

**Problem**: "Port 8000 in use"
```
Solution:
1. Kill existing: sudo pkill -f ktox_headless_loki
2. Change port in loki_engine.py
3. Verify port free: sudo netstat -tlnp | grep 8000
```

**Problem**: "Process crashed"
```
Solution:
1. Check logs: tail -f /root/KTOx/loot/loki/logs/loki.log
2. Verify dependencies: apt-get install nmap python3-pil
3. Check Python version: python3 --version (need 3.7+)
4. Permissions: sudo chown -R root:root /root/KTOx/loot/loki
```

**Problem**: "Timeout during startup"
```
Solution:
1. Increase timeout in loki_engine.py (line ~274)
2. Check for errors in logs
3. Verify network connectivity
4. Try headless operation (no LCD)
```

### WebUI Issues

**Problem**: "Cannot connect to WebUI"
```
Solution:
1. Verify running: netstat -tlnp | grep 8000
2. Check firewall: sudo ufw allow 8000
3. Use device IP not localhost
4. Try from same device first: http://127.0.0.1:8000
```

**Problem**: "WebUI slow/unresponsive"
```
Solution:
1. Monitor resource usage: top
2. Check logs for errors
3. Restart service: KEY1 to stop, KEY3 to start
4. Reduce scan timeout
```

### Data Issues

**Problem**: "No data saved to loot"
```
Solution:
1. Verify directory: ls -la /root/KTOx/loot/loki/
2. Check permissions: sudo chmod -R 755 /root/KTOx/loot/
3. Check available space: df -h
4. Review logs for errors
```

## Process Management

### Manual Control

**Start Loki manually:**
```bash
LOKI_DATA_DIR=/root/KTOx/loot/loki \
BJORN_IP=$(hostname -I | awk '{print $1}') \
LOKI_PID_FILE=/root/KTOx/loot/loki.pid \
python3 /root/KTOx/vendor/loki/ktox_headless_loki.py
```

**Stop Loki manually:**
```bash
pkill -f ktox_headless_loki
# Or via PID file:
kill $(cat /root/KTOx/loot/loki.pid)
```

**Monitor Loki:**
```bash
tail -f /root/KTOx/loot/loki/logs/loki.log
ps aux | grep loki
netstat -tlnp | grep 8000
```

### Systemd Integration (Optional)

Create `/etc/systemd/system/loki.service`:
```ini
[Unit]
Description=Loki Autonomous Security Engine
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/KTOx/vendor/loki
Environment="LOKI_DATA_DIR=/root/KTOx/loot/loki"
Environment="LOKI_PID_FILE=/root/KTOx/loot/loki.pid"
ExecStart=/usr/bin/python3 /root/KTOx/vendor/loki/ktox_headless_loki.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Then:
```bash
sudo systemctl daemon-reload
sudo systemctl enable loki
sudo systemctl start loki
sudo systemctl status loki
```

## Performance Notes

### Resource Usage
- **CPU**: 2-5% idle, 10-30% during scans
- **Memory**: ~200-300MB baseline, up to 500MB during attacks
- **Disk**: ~500MB for installation, grows with data
- **Network**: Depends on scan/attack intensity

### Optimization
- Run scans during off-hours
- Monitor `/root/KTOx/loot/loki/output/` size
- Archive old results periodically
- Use targeted scans vs network-wide

## Security Considerations

### Responsible Use
- Only test networks you own or have authorization for
- Respect privacy and confidentiality
- Document all testing activities
- Follow responsible disclosure

### Data Protection
- Loot data contains sensitive information
- Store securely or delete after testing
- Consider encryption for long-term storage
- Restrict file permissions: `chmod 700 /root/KTOx/loot/loki`

### Network Impact
- Scans can trigger IDS/WAF alerts
- Aggressive settings may cause service disruptions
- Coordinate with network administrators
- Use low-speed modes in production environments
