# SDR Suite Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a separate HackRF SDR Suite service for KTOX_Pi on port 8081 and link it from the existing WebUI.

**Architecture:** The SDR Suite is isolated from `web_server.py` and owns its own Flask + SocketIO server, static frontend, HackRF command wrapper, processing helpers, and capture database. The existing KTOX WebUI receives only a lightweight link to open the separate service.

**Tech Stack:** Python 3, Flask, Flask-SocketIO, SQLite, NumPy with graceful fallback handling, HackRF CLI tools, vanilla HTML/CSS/JS.

---

## File Structure

- Create `sdr/__init__.py`: package marker and version string.
- Create `sdr/device.py`: HackRF CLI wrapper and subprocess lifecycle management.
- Create `sdr/processing.py`: FFT, waterfall, spectrum normalization, and peak detection.
- Create `sdr/database.py`: SQLite capture metadata store.
- Create `sdr/handlers.py`: capture file helpers, preset frequencies, sweep parsing, and capture metadata assembly.
- Create `services/sdr_server.py`: Flask + SocketIO app on port 8081.
- Create `static/sdr/index.html`: standalone SDR Suite page.
- Create `static/sdr/js/api.js`: fetch helpers.
- Create `static/sdr/js/waterfall.js`: canvas waterfall renderer.
- Create `static/sdr/js/app.js`: UI state and SocketIO behavior.
- Create `static/sdr/css/style.css`: KTOX-compatible dark interface.
- Create `scripts/ktox-sdr.service`: systemd unit template.
- Create `tools/validate_sdr_suite.py`: no-hardware validator.
- Modify `requirements.txt`: add `numpy`, `flask-socketio`, and `python-socketio`.
- Modify `web/index.html`: add SDR Suite navigation link.
- Modify `web/app.js`: derive the SDR link host from the current page.
